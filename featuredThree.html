Library Inventory Management

Java project used as an inventory manager for a library/bookstore's novels

On the left you see the interface the user will see and the 10 options they are given. These 10 options decide the paths of execution the code will take, most prompts leading to more inputs neccessary from the user. The code does error checking of inputs to allow for only valid inputs to be accepted. Then as valid inputs are set for actions like saving a new book to the system, the code will save that book and its associated values for name, id, title, author, price, and count to the array of all novels, the inventory. This inventory can be viewed by the user, interacted with, and changed depending on the action that needs to be done.

The main connection I feel with this project and the capstone is that this project was the real beginning of my understanding of error handling with code and user inputs handling. In our capstone project we are constantly getting actions from the user and this handling of wrong inputs learning began with this project back in CISC 230.
